# 优化分视颜色校准与对齐测试提示 Spec

## Why
当前“对齐测试（AlignmentExercise）”的提示语对“哪条线该和哪条参考线平行”的描述不够具体，且参考线颜色未明确与用户实际 3D 眼镜颜色配置（左红右青 / 左青右红）绑定，容易导致用户理解偏差。
另外，系统现有的“镜片颜色选择/确认”与“分视对比度测试（暗光惩罚阈值）”之间缺少一个统一的“颜色强度/偏好”校准环节，导致后续所有分视类页面（检查与训练）缺少一致的颜色管线，难以把“镜片颜色 → 主观强度 → 惩罚梯度”串成闭环。

## What Changes
- 新增一个“颜色域校准”页面：在用户确认 3D 眼镜左右镜片颜色后，进入一个由红域/青(蓝)域组成的田字格矩阵，让用户选择“自己感受最强”的颜色域，并选定一个强度档位（梯度）。
- 在全局设置中新增“颜色域校准结果”状态，并提供统一的“分视颜色计算”出口，使后续所有分视类页面只依赖一个颜色来源。
- 优化对齐测试提示语：
  - 仅提示“平行”这一核心动作；
  - 动态显示“哪种颜色的线需要与哪种颜色的背景参考线平行”（基于用户镜片配置与颜色域校准结果生成）。
- 优化对齐测试的颜色/参考线逻辑：
  - 背景参考线颜色与“左眼可见颜色/右眼可见颜色”绑定，避免出现颜色与镜片配置相反的情况；
  - 参考线与操作线保持“同色系但不同亮度/透明度”，既能表达对应关系，又避免混淆。
- 串联暗光惩罚逻辑：
  - 如果触发“分视对比度测试”，其产出的惩罚因子继续作为最终颜色输出管线的一部分；
  - 后续所有分视训练/检查均使用“颜色域校准结果 × 惩罚因子”的最终颜色。
- 用户信息页展示：在 User Profile 中展示用户测得的校准颜色和最终颜色状态。
- 测试保障：编写单元测试验证状态管理中的颜色管线逻辑，并编写端到端(E2E)测试验证完整的“镜片选择 -> 颜色校准 -> 抑制测试 -> 对齐测试”流程中颜色的正确传递。
- 理论页补充：在“论文与原理”中补齐“镜片颜色确认 → 颜色域校准 → 抑制判定/惩罚阈值 → 对齐/训练”的系统链路说明，并补充对应的经典临床/论文关键词（如 dichoptic / binocular rivalry / anaglyph / luminance balancing）。

## Impact
- Affected specs: 分视检查与分视训练的一致性、可理解性与可复现性增强；减少用户因颜色映射错误造成的误操作。
- Affected code: `src/views/exercises/AlignmentExercise.vue`、3D 眼镜选择/确认相关页面、`src/store/settings.ts`、流程控制（`flowConfig`/`useFlowManager`）以及使用分视颜色的训练/检查页面。

## ADDED Requirements
### Requirement: 颜色域校准页面
系统 SHALL 在用户完成 3D 眼镜左右镜片确认后，展示“颜色域校准”页面，允许用户选择其主观感受最强的颜色域与强度档位，并持久化保存。

#### Scenario: 初次进入分视流程
- **WHEN** 用户完成镜片确认并进入下一步
- **THEN** 系统展示田字格颜色矩阵，用户可选“红域”或“青(蓝)域”及其强度档位，并进入后续检查/训练流程

### Requirement: 统一的分视颜色输出
系统 SHALL 使用统一的颜色输出规则给所有分视类页面提供颜色值：
- 最终颜色 = 镜片基础颜色 × 颜色域强度因子 ×（若需要）暗光惩罚因子

#### Scenario: 触发暗光惩罚后的统一显示
- **WHEN** 用户在分视对比度测试中被测得惩罚因子
- **THEN** 后续分视训练/对齐测试的颜色显示自动使用该惩罚因子，无需每个页面单独处理

## MODIFIED Requirements
### Requirement: 对齐测试提示语与参考线
系统 SHALL 在对齐测试第 2 步仅提示“平行”操作，并明确展示“哪种颜色的线与哪种颜色的背景参考线平行”，且参考线颜色与镜片配置保持一致。
